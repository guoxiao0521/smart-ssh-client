# Smart SSH Client Task List

## 项目现状

当前项目主线能力已经打通，是一个基于 Electron + Vue 的桌面 SSH 客户端。

已具备的核心链路：

- 读取本机 `~/.ssh/config`
- 选择主机并建立 SSH 连接
- 支持 `ProxyJump` 多跳连接
- 通过 SFTP 浏览远程目录
- 预览远程文件内容
- 打开远程终端并交互

当前产品更接近“可用的只读 SSH 客户端”，主打连接、浏览、预览、终端四条主线，文件管理和主机管理能力还没有补齐。

## 已完成功能

### 连接能力

- 解析本机 `~/.ssh/config`
- 支持读取 `Host`、`HostName`、`Port`、`User`、`IdentityFile`、`BindAddress`、`ProxyJump`
- 支持密钥、默认私钥、`ssh-agent` 认证
- 支持认证失败后密码回退
- 支持 `ProxyJump` 跳板链路连接
- 支持主动断开连接

### 文件能力

- 基于 SFTP 浏览远程目录
- 文件树采用导航式交互：点击目录进入该目录，顶部提供返回上级按钮
- 顶部路径输入框显示当前路径，支持手动输入后回车跳转，失焦自动恢复
- 文件和目录已做基础排序
- 支持打开文件预览弹窗
- 支持文本文件预览
- 支持代码高亮
- 支持图片预览
- 支持二进制文件提示
- 支持大文件保护，超过 `5MB` 不直接加载
- 支持文件下载（右键菜单下载，保存对话框选路径）

### 终端能力

- 支持创建远程 shell 会话
- 支持终端输入回传
- 支持终端窗口 resize
- 支持终端区域自适应
- 支持 Web 链接识别

### 桌面应用能力

- 已完成 Electron 主窗口集成
- 已完成预加载脚本与 IPC 通信桥接
- 已完成基础暗色 UI
- 已统一接入图标库并替换渲染层内联图标
- 已支持侧边栏折叠
- 已有基础错误提示与加载态
- 已接入基础构建与 Windows 发布流程

## 已知缺口

### 高优先级

- 目前前端只有单活动连接模型，尚不支持同时管理多个连接
- 文件能力仍缺少编辑、保存、重命名、新建目录（上传、下载、删除已支持）
- 主机列表只能读取 `~/.ssh/config`，不能在应用内新增、编辑、删除主机

### 中优先级

- 缺少连接状态恢复、重连策略和更清晰的错误分类
- 缺少文件树搜索、收藏路径、最近访问等效率功能
- 文件预览能力还不够完整，缺少更多格式识别和更友好的降级策略
- 终端还缺少标签页、历史会话和更完整的快捷键体验

### 低优先级

- 项目中仍有模板残留文件和占位配置
- 应用标题、项目描述、发布配置还可以进一步产品化
- 缺少自动化测试和验收文档

## 最建议优先做的 3 件事

> 综合实现成本与实用价值，以下三项最值得立即推进：

1. ~~**文件下载**（P0）~~ ✅ 已完成
2. **多连接模型**（P0）— 目前只能同时操作一台主机，升级为多标签/多连接后实用性质变，是专业 SSH 客户端的基础门槛。
3. **主机管理 UI**（P2 → 建议提前）— 现在只能手改 `~/.ssh/config`，在 GUI 内增删改主机是普通用户能独立使用的前提。
